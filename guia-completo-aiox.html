01 O que e o AIOX

O AIOX (AI-Orchestrated System) e um meta-framework que orquestra agentes de IA para desenvolvimento full-stack.

Em vez de voce fazer tudo sozinho com o Claude Code, voce ativa agentes especializados que tem personalidade, regras, heuristicas e processos proprios.


💻

Motor de execucao
+
🤖

Motoristas especializados
+
👥
Squads
Equipes completas
=
🚀
AIOX
Orquestracao inteligente
💡
Analogia: Claude Code e o motor do carro. Agentes sao motoristas especializados (piloto de F1, taxista, caminhoneiro). Squads sao equipes completas com varios motoristas.

03 Comandos: / vs * vs @

Os 3 prefixos que controlam tudo no AIOX.

/
Skill
Liga um agente ou capacidade
🟢 Usar no inicio, para ativar
*
Comando
Executa acao dentro do agente ativo
🟢 Depois que o agente esta ativo
@
Mencao
Chama/referencia outro agente
🟢 Para handoff ou referencia

Fluxo na pratica

1️⃣
Ativar
/skill ou @agente
➔
2️⃣
Greeting
Confirmacao visual
➔
3️⃣
Comandos
*help, *comando
➔
4️⃣
Trocar
@outro ou /skill
✅ CERTO
/squad-creator-pro:pedro-valerio
*eng-map agendamento-whatsapp

Ativa, depois usa comando. Pedro executa com heuristicas e veto.

❌ ERRADO
"cria uma automacao de agendamento"

Fala direto sem ativar. Claude Code executa generico.

06 Squad vs Agente — Qual Chamar?

Uma das duvidas mais comuns: qual a diferenca entre chamar o squad e chamar o agente direto?

🤖
Chamar um AGENTE

Ativa UM especialista com persona, comandos e DNA. Voce fala direto com ele.

/squad-creator-pro:pedro-valerio
➔ Pedro ativa, greeting aparece
➔ Voce usa *comandos dele
👥
Chamar o SQUAD

Ativa o CHIEF do squad, que decide qual agente usar e orquestra o time.

/squad-creator-pro:squad-chief
➔ Chief ativa, analisa seu pedido
➔ "Vou chamar o Alan pra isso"

Quando usar cada um

SituacaoUseExemplo
Voce sabe qual especialista precisaAgente direto/squad-creator-pro:pedro-valerio
Voce nao sabe por onde comecarSquad chief/squad-creator-pro:squad-chief
Precisa de varios agentes do squadSquad chiefChief orquestra a sequencia
Quer fazer algo especificoAgente + *comando*eng-map agendamento
🩹
Analogia: Chamar o agente = ligar direto pro cirurgiao. Chamar o squad = ligar pra recepcao do hospital e ser direcionado.
⚠️
Cuidado: O chief tambem pode diluir em conversas longas (e ate mais rapido que um agente direto, pois gerencia multiplos contextos). Use o chief para PLANEJAR, nao para EXECUTAR tudo numa conversa so.

09 A Verdade Nua e Crua

Como os agentes REALMENTE funcionam por baixo dos panos. Entender isso muda tudo.

Todos rodam no mesmo Claude Code

Agentes NAO sao processos separados. NAO sao containers. E uma unica IA que carrega personas diferentes — como trocar de roupa.

O que PARECE
Chief → Alan → Pedro
3 IAs separadas trabalhando
Cada uma com seu processo
O que REALMENTE E
1 Claude Code
1 contexto compartilhado
Trocando de "roupa" (persona)

O que isso significa na pratica

ConsequenciaImpactoSolucao
Contexto e finito e compartilhadoConversa longa = agente dilui1 conversa por agente
Chief nao "chama" agente de verdadeEle troca de roupa, nao inicia processoChief planeja, voce executa em conversas separadas
Troca de agente custa contextoCada troca carrega nova persona (~3-5K tokens)Handoff protocol compacta (~379 tokens)
Nao tem barreira entre agentesClaude Code pode responder "como ele mesmo"Use *help para verificar
⚙️
Pedro Valerio "E o caminho errado sendo possivel. O sistema nao bloqueia. Voce CONSEGUE continuar falando achando que tem alguem ali. Ate isso ser corrigido, a regra e: 1 conversa, 1 agente, 1 objetivo."
🧠
Alan Nicolas "E limitacao da tecnologia, nao do design. Quando tivermos multi-agente de verdade com processos separados, o chief vai orquestrar de fato. Hoje, ele planeja. A execucao e sua."

10 Stories — O Bastao Entre Conversas

A story e o mecanismo que carrega contexto entre conversas. Sem ela, voce repete tudo a cada troca de agente.

❌ Sem Story
Conversa 1: Chief planeja → voce lembra de cabeca
Conversa 2: Alan extrai → voce explica tudo de novo
Conversa 3: Pedro constroi → voce explica DE NOVO
Retrabalho toda vez
✅ Com Story
Conversa 1: Chief planeja → cria story com tudo
Conversa 2: Alan abre story → sabe o que fazer
Conversa 3: Pedro abre story → ve o que Alan fez
Zero retrabalho

Como uma story se parece

# Story 1.1 — IA de Agendamento WhatsApp

## Status: In Progress

## Objetivo
Criar IA que agenda consultas via WhatsApp integrada com CRM

## Acceptance Criteria
[x] Conhecimento extraido (Alan)
[x] Processo mapeado (Pedro)
[x] PRD escrito (@pm)
[ ] Arquitetura definida (@architect)
[ ] API implementada (@dev)
[ ] Testes passando (@qa)
[ ] Deploy feito (@devops)

## Notas
- Gap de 4h entre pedido e confirmacao
- Processo atual 100% manual por telefone

## File List
- docs/prd/ia-agendamento.md
- workspace/.../products/ia-agendamento/

12 10 Boas Praticas Obrigatorias

Siga essas regras e o ecossistema trabalha a seu favor.

01
Sempre ative antes de pedir
Sem ativacao, e Claude Code generico. Use /skill ou @agente primeiro.
02
Use *help como bussola
Chegou num agente? *help. E o menu do restaurante.
03
Um agente por conversa
5 agentes na mesma conversa = contexto diluido. Uma conversa, um foco.
04
Na duvida, reative
Conversa longa? *help. Resposta generica? Reativa com /skill.
05
Siga o fluxo dos agentes
@pm ➔ @architect ➔ @dev ➔ @qa ➔ @devops. Nao pule etapas.
06
*comandos > falar generico
*auto-rules crm ativa heuristicas. "Faz automacao" ativa nada.
07
Use stories como bastao
A story carrega contexto entre conversas. Cada agente le, executa e atualiza.
08
Chief planeja, voce executa
Use o squad chief pra planejar. Depois chame agentes diretos em conversas separadas.
09
Credenciais: cheque memoria
O sistema ja pode ter suas API keys salvas. Nao digite de novo.
10
Handoff explicito
"Terminei com Alan, agora Pedro." O agente nao sabe que voce saiu.

13 Erros Comuns e Como Evitar

Os 8 erros mais frequentes que todo usuario comete no inicio.

❌ Erro 1: Falar direto sem ativar agente
❌ Errado

"Cria uma API de agendamento"

Claude Code faz sem persona, sem regras.

✅ Certo

/AIOX:agents:dev ➔ *develop story-agendamento

Dev faz com quality gates e padroes.

❌ Erro 2: Nao perceber que o agente "saiu"
❌ Errado

Conversa longa ➔ agente diluiu ➔ continua achando que ta ativo

✅ Certo

Conversa longa ➔ *help ➔ ve se resposta tem persona

❌ Erro 3: Pular etapas do fluxo
❌ Errado

Ideia ➔ @dev direto = codigo sem PRD, sem arquitetura

✅ Certo

Ideia ➔ @pm ➔ @architect ➔ @sm ➔ @po ➔ @dev

❌ Erro 4: Fazer tudo numa conversa so
❌ Errado

Uma conversa: PRD + arquitetura + codigo + testes + deploy

✅ Certo

Conversa por fase: cada uma com 1 agente e 1 foco

❌ Erro 5: Confundir SOP com Task
❌ Errado

Criar SOP para o @dev. SOP e para HUMANOS.

✅ Certo

Criar TASK para o @dev (inputs, outputs, veto).

❌ Erro 6: Dar comando generico ao agente
❌ Errado

"Me ajuda com processos" — vago demais

✅ Certo

*eng-map atendimento-clinica — modo especifico

❌ Erro 7: Achar que o chief executa tudo sozinho
❌ Errado

Chief orquestrando 5 agentes numa conversa so. Contexto estoura.

✅ Certo

Chief planeja (1 conversa). Voce executa com agentes diretos (conversas separadas).

❌ Erro 8: Trocar de conversa sem story
❌ Errado

Abre nova conversa e explica tudo do zero pro novo agente.

✅ Certo

"Le a story docs/stories/1.1.story.md" — agente ja sabe tudo.

16 Glossario

Todos os termos tecnicos do ecossistema AIOX.

Agent
Especialista IA com persona, voz e heuristicas proprias
Greeting
Mensagem de apresentacao do agente (confirma que esta ativo)
Handoff
Transicao estruturada de um agente para outro
Heuristica
Regra de decisao do agente (SE X, ENTAO Y)
Persona
Identidade do agente (nome, estilo, crencas)
PRD
Product Requirements Document (o que o produto faz)
Quality Gate
Checkpoint de qualidade (PASS/FAIL)
Skill
Atalho / para ativar agente ou capacidade
SOP
Standard Operating Procedure (procedimento para humanos)
Squad
Time de agentes organizado por dominio
Squad Chief
Lider do squad que orquestra os agentes e planeja
Story
Unidade de trabalho com objetivo, ACs e checkboxes de progresso
Task
Instrucao estruturada que um agente IA executa
Veto Condition
Regra que BLOQUEIA aprovacao se nao for atendida
Voice DNA
Padrao de comunicacao extraido (como a pessoa fala)
Thinking DNA
Padrao de pensamento extraido (como a pessoa decide)
Workflow
Sequencia de tasks conectadas com checkpoints
Workspace
Dados da empresa (marca, produtos, credenciais)
Pareto ao Cubo
0,8% genialidade, 4% excelencia, 20% impacto, 80% eliminar
Diluicao
Quando o agente perde contexto e o Claude Code responde generico
Memoria
Sistema persistente que guarda credenciais, preferencias e decisoes entre conversas
